Версия конфигурации

Показывается номер текущей версии типовой конфигурации. Номер версии изменяется автоматически при обновлении конфигурации, вручную он не редактируется. Процесс обновления конфигурации можно запустить по ссылке Информация об обновлениях.

Заголовок главного окна

При запуске программы в заголовке главного окна отображается наименование программы. Если в программе задана основная организация, то в заголовке главного окна отображается наименование  основной организации. При одновременной работе с несколькими информационными базами, удобнее заменить стандартный заголовок на такой, который поможет быстро определить, какая именно из информационных баз сейчас открыта. Для этого нужно ввести  текст в поле Заголовок. 

Обмен данными

Задаются параметры для автоматического обмена данными при работе с распределенными информационными базами, а также для обмена данными с конфигурацией "1С:Управление торговлей 8".

Префикс узла для распределенной информационной базы - префикс задается, если предполагается обмен данными между распределенными информационными базами (РИБ). Префиксация позволяет избежать дублирования кодов элементов в узлах РИБ.

Префиксация осуществляется только если в списке узлов плана обмена Полный создан хотя бы один узел. В этом случае префикс добавляется к кодам при создании новых элементов справочников, документов и элементов планов видов характеристик. Установка или изменение префикса не приводит к перенумерации кодов уже существующих элементов.

Примечание
Не рекомендуется в префиксах использовать цифры. Это может привести к ошибочной нумерации объектов или к ошибкам при обмене данными.

Пользователь, в клиентском сеансе которого должны выполняться фоновые и регламентные задания в файл-серверном режиме работы информационной базы, задается в поле Пользователь. Рекомендуется запускать отдельный клиентский сеанс от имени указанного пользователя. Это позволяет производить автоматический обмен по расписанию. При этом пользователь должен обладать достаточными правами для выполнения произвольных операций по чтению и записи данных информационной базы. Для клиент-серверного режима работы этот параметр задавать не требуется.

Интервал опроса регламентных заданий в файловом режиме необходимо задавать только для файл-серверного режима работы информационной базы.

Склад для обмена данными с конфигурацией "Управление торговлей" необходимо указывать только при использовании обмена с конфигурацией "Управление торговлей". При загрузке данных во всех загружаемых документах устанавливается указанный склад.

Права доступа

Если с информационной базой работают несколько пользователей, и при этом в базе ведется учет нескольких организаций, часто необходимо ограничить доступ отдельных пользователей к данным определенных организаций (разрешить просмотр данных по организации, но запретить изменять данные, либо запретить просмотр и изменение данных).

Для использования такой возможности необходимо установить флажок Ограничить доступ к данным по организациям. После установки флажка необходимо применить данную настройку по кнопке ОК и перезапустить программу. Затем необходимо снова открыть настройку программы на этой закладке и нажать на кнопку Настроить права доступа. В открывшемся списке организаций нужно настроить права доступа, используя подменю Права верхней командной панели списка.

Запрет изменения данных

Можно установить дату, ранее которой (включая саму дату запрета изменения данных) будет запрещено изменять любые данные в информационной базе. Дату запрета можно установить по кнопке Установить дату запрета. Дату запрета можно установить одну для всей информационной базы, либо установить разные даты запрета для разных организаций и пользователей.

Пользователи, имеющие полные права, по умолчанию могут изменять данные ранее даты запрета. Если требуется запретить изменять данные и этим пользователям тоже, необходимо установить флажок Распространить дату запрета на пользователей с полными правами.

Каталог дополнительной информации

Указать каталог на диске, в котором хранятся внешние обработки и документы, полученные средствами встроенного обозревателя, можно в поле Каталог.

Контроль времени

Можно включить (отключить) проверку отличия во времени на сервере и клиенте (флажок Предупреждать о разнице времени с сервером 1С:Предприятия). Если флажок установлен, то в случае, если разница во времени на компьютерах сервера и клиента, составляет более 15 минут, выдается предупреждающее сообщение.

Контроль проведения

Можно включить (отключить) контроль последовательности проведения документов (флажок Контролировать последовательность проведения). Контроль последовательности проведения документов предназначен для исключения ошибок, связанных с изменением документов "задним числом".  Если флажок установлен, то при проведении регламентных операций закрытия месяца будет производиться контроль последовательности документов. Отключение контроля проведения может повысить производительность при многопользовательской работе.

Печать

Предусмотрена возможность настройки вывода артикула и кода номенклатуры в печатные формы. На закладке нужно задать настройку Дополнительная колонка печатных форм документов. Можно выбрать один из следующих вариантов:

  • Не выводить
    • Для регламентированных форм в колонке, предназначенной для вывода кода номенклатуры, выводится код справочника, например, для форм ТОРГ-12, М-11;
    • Для не регламентированных форм никаких колонок в печатные формы не добавляется;
  • Артикул
    • Для регламентированных форм артикул выводится в колонке, предназначенной для вывода кода номенклатуры;
    • Для не регламентированных форм добавляется колонка "Артикул";
  • Код
    • Для регламентированных форм в колонке, предназначенной для вывода кода номенклатуры, выводится код справочника;
    • Для не регламентированных форм добавляется колонка "Код".
